1 INTRODUÇÃO

        Esta atividade tem o objetivo de aprimoramento dos conceitos envolvidos na teoria de sistemas especialistas;

        Inúmeros problemas em IA necessitam de um método de busca, como, por exemplo, um jogo da velha inteligente, onde o sistema deve analisar as possibilidades e identificar a melhor delas. Se o sistema tivesse que analizar todas as possíveis combinações iria elevar o custo computacional, com isso foram desenvolvidos métodos de busca inteligente, o A* foi o utilizado nessa atividade.

        O algoritmo A* é uma extensão do algoritmo de Dijkstra, onde se analisa a heurística afim de encontrar o melhor caminho entre dois nós de um grafo.

2 REFERENCIAL TEÓRICO

        O algoritmo A* consiste em um algoritmo BFS, onde o nó de um grafo que contém o menor valor de heurística para o nó de chegada seja expandido primeiro.

2.1 Algoritmo A*

        Este algoritmo é aplicável quando se dispõe de uma função de avaliação heurística do custo, denotada f’(·). Em muitas aplicações, o custo incorrido ao percorrer, a partir de S, um certo caminho até o nó N, pode ser calculado através de uma função g(·). Se for disponível uma função heurística h’(·) que permite estimar o custo para percorrer a árvore de N até o nó objetivo G (goal), então, f’(N) = g(N) + h’(N). (Meneghetti, 2004)

        O A* pode ser aplicado em qualquer situação que possa ser representada por um grafo ou árvore, como, por exemplo, o labirinto objetivo deste relatório. O algoritmo A* base está na Figura 1.

3 METODOLOGIA

        O labirinto consiste em um ponto de partida, um de chegada, casas livres e obstáculos, o objetivo é encontrar o melhor caminho entre a partida e a chegada do labirinto passando somente pelas casas livres. O labirinto possui um custo vertical, horizontal e diagonal dependente do movimento.

        Para representar a partida foi utilizado no labirinto a cor azul, para representar a chegada a cor vermelha, a cor branca para casas livres, a cor verde para o caminho encontrado e a cor ciano para as casas visitadas pelo A*.

4.2 Resultado para o Custo Diagonal Elevado

        Com o custo diagonal de valor de 200 (muito maior que o valor original 1), com isso o A* tende a escolher as casas livres que não estão na diagonal das casas em que ele analisa, com isso, os movimentos diagonais passam a acontecer somente quando necessário. 5 CONCLUSÃO

        O uso do algoritmo A* como decisor para encontrar o melhor caminho no labirinto é de grande valia, pois é um método de busca inteligente, que ajuda a diminuir consideravelmente o custo computacional, além de ser bastante eficaz em encontrar a solução desejada.

        A variação do custo diagonal causa uma elevação no custo computacional, já que mais casas são visitadas, mas  nada exagerado, com isso, o método continua útil e aplicável.
